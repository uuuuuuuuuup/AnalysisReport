# 稳健投资策略分析助手 v2.1

## 技能描述

本技能实现稳健投资策略框架，通过**主代理+子代理**的多代理架构对股票进行深度价值分析，并按数据类型接入东方财富 mx-* skills。适用于：
- 用户请求分析某只股票
- 用户询问是否应该买入某只股票
- 用户需要估值分析和安全边际评估

---

## 架构设计

### 角色分工

**主代理（Coordinator）**
- 输入解析与任务调度
- 创建输出目录
- 并行启动子代理（使用 Agent 工具）
- 等待子代理写出数据包文件后汇聚结果
- 执行因子分析（Phase 3）
- 生成最终报告

⚠️ **主代理不直接调用任何数据采集工具，所有数据采集由子代理完成**

**子代理 1（Phase 1 Worker）**
- 采集市场数据：股价、三大报表、股息历史、K 线、无风险利率、管理层信息、行业竞争格局
- 输出：`data_pack_market.md`

**子代理 2（Phase 2 Worker）**
- 采集深度财务数据：报表附注、审计报告、关联方交易、非经常损益、研报
- 输出：`data_pack_report.md`

### 工作流程图

```
用户请求
    ↓
主代理：输入解析 + 目标年份确定
    ↓
主代理：mkdir 创建输出目录
    ↓
主代理：Agent 工具（并行启动两个子代理）
    ├─→ 子代理 1（Phase 1）→ data_pack_market.md
    └─→ 子代理 2（Phase 2）→ data_pack_report.md
    ↓
主代理：Read 工具轮询，等待两个文件生成
    ↓
主代理：读取数据包，验证完整性
    ↓
主代理：执行因子分析（1A → 1B → 2 → 3 → 4）
    ↓
主代理：写出最终报告
```

## 触发条件

当用户输入包含以下任一情况时，立即调用本技能：
1. 股票代码或名称（如"分析平安银行"、"000001"、"长和"）
2. 询问估值、安全边际、是否值得买入等问题
3. 提到"稳健投资策略"或"turtle strategy"

## 主代理执行流程

### 第一阶段：输入解析

从用户消息中提取：
1. **股票代码/名称**：识别目标股票
2. **持股渠道**：港股通/直接持有/美股券商（未指定则用默认值）
3. **目标年份**：
   - 当前月份 ≤ 3 月 → target_year = 当前年份 - 2
   - 当前月份 ≥ 4 月 → target_year = 当前年份 - 1

### 第二阶段：创建输出目录

```bash
mkdir -p {workspace}/稳健投资策略分析报告/{symbol}/
```

### 第四阶段：监控子代理状态

主代理使用 Read 工具检查输出文件是否生成：
- 检查 `data_pack_market.md`：存在且非空 → Phase 1 完成
- 检查 `data_pack_report.md`：存在且非空 → Phase 2 完成
- 若 data_pack_market.md 缺失超过合理时间 → 通知用户，可选择重试
- data_pack_report.md 可选，若缺失则使用降级方案继续分析

### 第五阶段：数据包整合

```
1. 读取 data_pack_market.md
2. 检查文件头部单位标注（百万元 / 千元 / 万元）
3. 读取 data_pack_report.md（若存在）
4. 检查是否存在 2.5 节（中间期财报摘要）
5. 若存在中间期数据，临时存储以备预警使用（不用于核心估值）
```

## 因子分析框架

> 本章节由主代理在 Phase 3 执行，基于两个子代理产出的数据包进行分析。

### 因子 1A：五分钟快筛（一票否决）

检查 6 项，任一项为"是" → 否决：
1. 审计意见异常
2. 频繁更换审计师
3. 财务造假或重大违规前科
4. 看不懂（无法理解商业模式）
5. 商业模式未被验证
6. 控股股东重大负面信号

### 因子 1B：深度定性分析

**必须完成的参数预提取**：

| 类别 | 参数 |
|:---|:---|
| 盈利 | 归母净利润、折旧摊销、资本开支、SBC |
| 现金 | 狭义现金、广义现金、受限现金、有息负债、广义净现金 |
| 分配 | 支付率锚定值、年均回购金额、股息税率 |
| 估值 | Rf、门槛值 = max(3.5%, Rf+2%) |

**定性模块**：资本消耗强度、收款模式、护城河（两层递进）、周期性、人力资本依赖、管理层治理、监管风险、MD&A 解读、控股折价（如适用）

### 因子 2：穿透回报率粗算（Top-Down）

**公式**：`粗算穿透回报率 = [Owner_Earnings × 支付率锚定值 × (1−股息税率) + 年均回购] / 市值`  
其中：`Owner_Earnings = 归母净利润 + 折旧摊销 − 维持性资本开支`（维持性资本开支 = 折旧摊销 × 维持性Capex系数，系数通常取0.6-0.8）

**否决门**：
- R < Rf → 检查 EV 口径，若仍不通过则否决
- R < 门槛值×0.5 → 检查 EV 口径，若仍不通过则否决
- 门槛值×0.5 ≤ R < 门槛值 → 标注"边际不达标"，继续进入因子 3

### 因子 3：精算穿透回报率（Bottom-Up）+ 现金质量审计

**核心原则**：极端保守假设，逐行追踪现金流底层

**计算要点**：
1. **真实现金收入** = 营业收入 − max(0, 应收变动) − max(0, −预收变动)
2. **非经常项分类**：保留 V1（资产处置）+ V5（投资收入）；扣除 V2（补贴）+ V3（保险）+ V4（其他一次性）
3. **经营现金支出**：优先从现金流量表直接取数（购买商品/员工/税费/利息）；若明细不足，采用 OCF 倒推法
4. **资本扣除**：Capex全额 + 对外投资 + 隐性必要支出
5. **真实可支配现金结余** = 真实现金收入 + V1 + V5 − 应扣除非经常性流入 − 经营现金支出 − 资本扣除
6. **合理性校验**：真实可支配现金结余 与 FCF 比较，若 真实可支配现金结余 > FCF×1.5 则需修正
7. **现金储备质量**：可自由支配现金 = 广义现金 − 受限现金 − 上游障碍现金 − 专项用途资金
8. **精算回报率**：`精算穿透回报率 = [真实可支配现金结余 × 支付率锚定值 × (1−股息税率) + 年均回购] / 市值`；若广义净现金/市值 > 40%，同时计算 EV 口径

**收入敏感性分析**：计算 0.9×、0.8×、0.7× 收入情景下的回报率

**外推可信度**：基于收入波动率、利润调整幅度、粗算偏差、经营模式变化、lambda 可靠性 5 维度评定

### 因子 4：估值与安全边际

**门槛值**：港股 max(5%, Rf+3%)，A股 max(3.5%, Rf+2%)，美股 max(5%, Rf+3%)

**价值陷阱排查**（5项）：
- 现金流趋势性恶化、护城河收窄、行业结构性衰退、分配意愿弱、管理层损害价值
- N≥2 且 精算穿透回报率 ≤ 门槛值×1.5 → 排除

**安全边际**：`修正后安全边际 = 精算穿透回报率 − 门槛值`（周期性修正：底部−1pct，顶部+2pct）

**现金保护等级**：
| 等级 | 广义净现金/市值 | 安全边际折扣 |
|:---|:---:|:---:|
| 无保护 | <20% | 30% |
| 轻度 | 20-40% | 25% |
| 强 | 40-60% | 20% |
| 极强 | >60% | 15% |

**目标买入价**：`目标买入价 = 当前股价 × (精算穿透回报率/门槛值) × 折扣系数`

## 注意事项

1. **职责边界**：主代理只做调度和分析，不调用数据采集工具；数据采集完全由子代理完成
2. **工具分工**：严格按数据类型选用工具（见可用工具清单中的适用场景）
3. **数据优先级**：
   - 无风险利率 → mx_macro_data 优先
   - 公告/研报 → mx_finance_search 优先
   - 定性信息/管理层/竞争格局 → mx_financial_assistant 优先
   - 实时行情/股息 → mx_finance_data 优先
   - 三大报表/K线 → AI-Tools MCP 优先
4. **并行执行**：Phase 1 和 Phase 2 子代理可以并行执行，主代理等待两个文件都生成后再继续
5. **数据标注**：所有数据必须标注来源（工具名称/搜索关键词）
6. **单位统一**：报告输出使用人民币亿元，子代理数据包使用报表原始币种百万元
7. **否决纪律**：因子 1A 或因子 2 触发否决 → 立即停止后续分析
8. **策略目录只读**：不得修改 V1.x.md 历史版本文件
9. **框架边界**：不适用于 Pre-profit 企业、高增长科技股、周期顶部的强周期股

## 特殊规则

### 合同负债计入类现金

适用条件（须同时满足）：先款后货模式、交付确定性>95%、近3年波动率<30%
- 用于：安全垫评估、EV 口径计算
- 不用于：可自由分配现金 FF

### EV 口径双轨制

触发条件：广义净现金/市值 > 40%
- EV = 市值 − 广义净现金
- 通过条件：R_EV ≥ II 且通过现金分配意愿检验
