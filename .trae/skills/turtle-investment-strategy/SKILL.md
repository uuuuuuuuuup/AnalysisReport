# 龟龟投资策略分析助手 v2.1

## 技能描述

本技能实现龟龟投资策略框架，通过**主代理+子代理**的多代理架构对股票进行深度价值分析，并按数据类型接入东方财富 mx-* skills。适用于：
- 用户请求分析某只股票
- 用户询问是否应该买入某只股票
- 用户需要估值分析和安全边际评估

---

## 架构设计

### 角色分工

**主代理（Coordinator）**
- 输入解析与任务调度
- 创建输出目录
- 并行启动子代理（使用 Agent 工具）
- 等待子代理写出数据包文件后汇聚结果
- 执行因子分析（Phase 3）
- 生成最终报告

⚠️ **主代理不直接调用任何数据采集工具，所有数据采集由子代理完成**

**子代理 1（Phase 1 Worker）**
- 采集市场数据：股价、三大报表、股息历史、K 线、无风险利率、管理层信息、行业竞争格局
- 输出：`data_pack_market.md`

**子代理 2（Phase 2 Worker）**
- 采集深度财务数据：报表附注、审计报告、关联方交易、非经常损益、研报
- 输出：`data_pack_report.md`

### 工作流程图

```
用户请求
    ↓
主代理：输入解析 + 目标年份确定
    ↓
主代理：mkdir 创建输出目录
    ↓
主代理：Agent 工具（并行启动两个子代理）
    ├─→ 子代理 1（Phase 1）→ data_pack_market.md
    └─→ 子代理 2（Phase 2）→ data_pack_report.md
    ↓
主代理：Read 工具轮询，等待两个文件生成
    ↓
主代理：读取数据包，验证完整性
    ↓
主代理：执行因子分析（1A → 1B → 2 → 3 → 4）
    ↓
主代理：写出最终报告
```

## 触发条件

当用户输入包含以下任一情况时，立即调用本技能：
1. 股票代码或名称（如"分析平安银行"、"000001"、"长和"）
2. 询问估值、安全边际、是否值得买入等问题
3. 提到"龟龟投资策略"或"turtle strategy"

## 主代理执行流程

### 第一阶段：输入解析

从用户消息中提取：
1. **股票代码/名称**：识别目标股票
2. **持股渠道**：港股通/直接持有/美股券商（未指定则用默认值）
3. **目标年份**：
   - 当前月份 ≤ 3 月 → target_year = 当前年份 - 2

## 因子分析框架

> 本章节由主代理在 Phase 3 执行，基于两个子代理产出的数据包进行分析。

#### 因子 1A：五分钟快筛（一票否决）

检查 6 项：
1. 审计意见异常
2. 频繁更换审计师
3. 财务造假或重大违规前科
4. 看不懂（无法理解商业模式）
5. 商业模式未被验证
6. 控股股东重大负面信号

任一项为"是" → 否决，停止全部后续分析

#### 因子 1B：深度定性分析

**模块〇：三大报表基础数据与关键指标（强制步骤）**

**⚠️ 核心原则：数据只取一次，来源只标一次，后续因子直接引用不再重新取数**

必须完成的子任务：
1. **利润表核心数据**（逐年列表，标注来源）
   - 营业收入、营业成本、毛利润
   - 研发费用、销售费用、管理费用
   - 经营利润、其他收入/支出、税前利润
   - 所得税、归母净利润、少数股东损益

#### 因子 2：穿透回报率粗算（Top-Down）

计算步骤：
1. 归母净利润
2. Owner Earnings 粗算
3. 分配能力验证（一票否决）
4. 现金上游障碍
5. 分配意愿
6. 可预测性
7. 税务泄漏修正
8. 粗算穿透回报率 R%

**粗算否决门**：
- 若 R < Rf → 检查 EV 口径，若仍不通过则否决
- 若 R < II×0.5 → 检查 EV 口径，若仍不通过则否决
- 若 II×0.5 ≤ R < II → 标注"边际不达标"，继续进入因子 3

#### 因子 3：穿透回报率精算（Bottom-Up）+ 现金质量审计

**核心原则**：极端保守假设，逐行追踪现金流底层

**计算步骤**：

**步骤 1：真实现金收入还原**
```
营业收入 = S
应收账款净变动 = T（增加为正，减少为负）
预收款/合同负债净变动 = U（增加为正）

保守处理：
- 应收账款净增加 → 扣除（收入未收到现金）
- 预收款净增加 → 不加回（对应未来交付义务）
- 例外：高确定性预收模式（白酒/物业）可将净增加加回

真实现金收入 = S - T 的净增加部分 - U 的净减少部分
收款比率 = 真实现金收入 / S
```

**步骤 2：应收账款 footnote 核查**（收款比率<1 时必做）
- 一年以上应收占比
- 坏账计提政策 vs 同行
- 关联方应收占比
- A 股收入确认激进度判断

**步骤 3：非经常性现金流入分类**
```
保留项（真实可分配现金，不扣除）：
- 资产处置收入 V1（逐项列示）
- 其他投资收入 V5（联营分红/利息/理财收益）

扣除项（非可分配，剔除）：
- 政府补贴 V2
- 保险赔款 V3
- 其他一次性非投资类流入 V4

应扣除合计 = V2 + V3 + V4
```

**步骤 4：经营性现金支出还原**
```
⚠️ 核心原则：必须直接从现金流量表取数，不得用利润表科目估算替代。

第一优先级：从现金流量表直接取以下行（data_pack 2.4 节）：
- 供应商支付 = "购买商品、接受劳务支付的现金"
- 员工支付   = "支付给职工及为职工支付的现金"
- 现金税款   = "支付的各项税费"
- 现金利息   = "支付利息" 或财务费用中的实际现金部分
  （若资本化利息 > 30% → 还原为当期支出）

经营性现金支出合计 W = 供应商支付 + 员工支付 + 现金税款 + 现金利息

第二优先级（当现金流量表未分项列示，员工支付标注"未披露"时）：
- 不得用营业成本 + 税款 + 利息估算 W（会系统性低估）
- 改用 OCF 倒推法：W = 真实现金收入 − OCF
  含义：令底层计算与已知 OCF 挂钩，避免遗漏支出项
- 标注：⚠️ 现金支出明细不足，已采用 OCF 倒推法估算 W，AA 结果偏保守
```

**步骤 5：资本开支与投资的极端保守处理**
```
核心原则：全部纳入扣除项，不区分维持性/成长性

(a) 资本开支总额 = E（全额扣除）
(b) 投资性购买 = X1（对外投资 + 收购 + 参股，逐项列示）
(c) 隐性必要支出还原 = X2
    - 研发费用资本化还原
    - 销售费用资本化还原
    - 递延资产费用化还原

总扣除额 Y = E + X1 + X2
```

**步骤 6：准则差异核查**
- A 股：收入确认激进度、递延税资产趋势、资本化利息比例
- 港股：HKFRS 9 预期信用损失、HKFRS 16 表外租赁
- 或有负债：对外担保、重大诉讼、资本承诺

**步骤 7：真实可支配现金结余**
```
真实可支配现金结余 = 真实现金收入
                    + 资产处置收入 V1
                    + 其他投资收入 V5
                    - 应扣除非经常性流入 V_deduct
                    - 经营性现金支出 W
                    - 总扣除额 Y

逐年列表计算
历史均值 AA = [值] 亿元
  - 若存在"非经营收入主导型年份"（保留项>经营现金流 50%）
  - 区分 AA_incl（含全部年份）和 AA_excl（剔除异常年份）
  - 两者差异>30% → 以 AA_excl 为基准

━━━ 合理性校验（必须执行，不可跳过）━━━

AA_fcf = OCF − Capex（直接来自现金流量表，不含 V1/V5 调整）

比较 AA 与 AA_fcf：
- 若 AA > AA_fcf × 1.5：
    → ⚠️ AA 高于 FCF 的 1.5 倍，W 很可能计算不完整（常见原因：员工支付未纳入）
    → 回到步骤 4，检查是否用了 OCF 倒推法
    → 若无法补全 W：取 AA_conservative = min(AA, AA_fcf × 1.2)，后续以此为基准
    → 标注："AA 已按 OCF×1.2 修正为 [值] 亿元"
- 若 AA ≤ AA_fcf × 1.5：校验通过，直接使用 AA
```

**步骤 8：现金储备质量验证**
```
账面现金（广义）BB = [值] 亿元
  - 狭义现金 BB_narrow = 现金 + 短投
  - 定期存款/理财 BB_deposit = 广义 − 狭义
  - 合同负债 BB_contract（类现金，仅用于安全垫和 EV 计算）

受限资金 CC = 质押/冻结/监管账户
上游障碍现金 DD = 内地子公司留存
专项用途资金 EE

可自由支配现金 FF = BB − CC − DD − EE
  - 默认使用狭义口径 FF_narrow
  - 可升级至广义口径的条件：
    (a) 定期存款剩余期限 ≤ 1 年
    (b) 无质押/冻结限制
    (c) BB_deposit / BB_narrow < 50%

派息可持续年限 = FF / 年派息总额
```

**步骤 9：派息后现金储备净变动**
```
逐年计算：
期末可自由支配现金 - 期初可自由支配现金 - 当年结余 + 当年派息及回购
判断：是否持续为正（造血>分配）
```

**步骤 10：精算穿透回报率计算**
```
选用基准值 AA = [AA_incl / AA_excl] 亿元
支付率锚定值 M = [值]%（与因子 2 同口径）

─── 主轨：市值口径 ───
精算穿透回报率 GG = [AA × M × (1 − Q%) + O] / 当前市值

─── 辅轨：EV 口径（仅当广义净现金/市值 > 40% 时）───
EV = 当前市值 − 广义净现金
精算穿透回报率 GG_EV = [AA × M × (1 − Q%) + O] / EV

决策口径判断：
- 若 GG ≥ II → 使用市值口径
- 若 GG < II 但 GG_EV ≥ II 且通过"现金分配意愿检验" → 使用 EV 口径
```

**步骤 11：收入敏感性分析**
```
经营杠杆系数 λ = 过去 3 年Δ结余/Δ收入的中位数
λ可靠性校验：
  - 收入最大波幅<10% → 可靠性低
  - Δ结余/Δ收入符号不一致 → 稳定性差
  - λ>3 或 λ<0 → 异常

收入情景分析：
| 情景 | 收入 | 预估结余 | 精算穿透回报率 | vs 门槛 | 门槛对应股价 |
| 1.0× | S | AA | GG | ±X pct | P_th |
| 0.9× | S×0.9 | AA' | GG' | ±X pct | P_th' |
| 0.8× | S×0.8 | AA'' | GG'' | ±X pct | P_th'' |
| 0.7× | S×0.7 | AA''' | GG''' | ±X pct | P_th''' |

临界收入倍数 = 临界收入 / S_current
  - ≥0.85 → 安全边际对收入波动敏感
  - 0.7-0.85 → 韧性中等
  - <0.7 → 韧性强
```

**步骤 12：与因子 1 交叉验证 + 外推可信度评级**
```
5 维度评定：
① 收入波动率（5 年 CV）：<10%高，10-25%中，>25%低
② 利润口径调整幅度：C_adj/C_rep偏差<5%高，5-15%中，>15%低
③ 粗算偏差|HH|：<1pct 高，1-3pct 中，>3pct 低
④ 经营模式变化：无重大变化高，有调整中，转型中低
⑤ λ可靠性：正常高，有一项警告中，多项警告低

评定规则：
- 5 项中≥4 项为"高" → 外推可信度 = 高
- 任意 2 项为"低" → 外推可信度 = 低
- 其余 → 外推可信度 = 中
```

#### 因子 4：估值与安全边际

**步骤 1：门槛计算**
```
Rf = [输入值]%

港股门槛 = max(5%, Rf + 3%) = [II]%
A 股门槛 = max(3.5%, Rf + 2%) = [II]%
美股门槛 = max(5%, Rf_US + 3%) = [II]%

─── 双轨门槛比较 ───
主轨（市值口径）：GG vs II
辅轨（EV 口径）：GG_EV vs II（若适用）

判断：
- GG ≥ II → 主轨通过
- GG < II 但 GG_EV ≥ II 且通过分配意愿检验 → EV 口径通过
- GG < II 且 GG_EV < II → 不达标，排除
```

**步骤 2：价值陷阱排查**（5 项检查）
```
逐项检查（调用前序因子结论）：

| 序号 | 陷阱特征 | 判断标准 | 结果 |
| 1 | 现金流趋势性恶化 | 因子 3 结余序列连续≥2 年下降且降幅>15% | [是/否] |
| 2 | 护城河正在收窄 | 因子 1B 模块三判定为"负面" | [是/否] |
| 3 | 行业结构性衰退 | 因子 1B 模块四终端需求不可逆萎缩 | [是/否] |
| 4 | 分配意愿存疑 | 因子 2 分配意愿判定为"弱" | [是/否] |
| 5 | 管理层损害价值 | 因子 1B 模块六判定为"损害价值" | [是/否] |

存在项数 N：
- N = 0 → 风险低
- N = 1 → 风险中
- N ≥ 2 → 风险高
  - 若 GG > II × 1.5 → 保留（回报率足够覆盖风险）
  - 否则 → 排除
```

**步骤 3：安全边际与仓位**
```
决策口径选择：
- 若因子 3 决策口径 = EV 口径 → 使用 GG_EV
- 否则 → 使用 GG

安全边际 JJ = 选用回报率 − II
周期性修正：
- 强周期底部 → 门槛下调 1pct
- 强周期顶部 → 门槛上调 2pct
- 非周期/周期中段 → 不调整

修正后安全边际 KK

现金保护等级判定：
- 广义净现金/市值 < 20% → 无保护（30% 折扣）
- 20-40% → 轻度保护（25% 折扣）
- 40-60% → 强保护（20% 折扣）
- > 60% → 极强保护（15% 折扣）

仓位矩阵：
| 条件 | 仓位 |
|------|------|
| 安全边际≥1.5pct + 外推可信度高 + 陷阱风险低 | 标准仓位 |
| 安全边际≥1.5pct + 外推可信度高 + 陷阱风险中 | 70% |
| 安全边际≥1.5pct + 外推可信度中 + 陷阱风险低 | 70% |
| 安全边际 0.5-1.5pct + 外推可信度高 + 陷阱风险低 | 50% |
| 安全边际 0.5-1.5pct + 外推可信度中 + 陷阱风险低 | 观察仓位（30%） |
| 安全边际 0-0.5pct | 观察仓位或等待 |
| 安全边际 < 0 | 不建仓（现金保护豁免除外） |
| 外推可信度低 | 观察仓位或不建仓 |
| 陷阱风险高且回报率<门槛×1.5 | 排除 |

现金保护豁免（仅当使用 EV 口径且安全边际≥0 时）：
- 若现金保护等级为"强保护"或"极强保护"，且市值口径安全边际<0
- → 升级为"高优先级观察"
- → 标注："EV 口径达标，等待市值口径收窄至达标"
```

**步骤 4：股价位置与买入触发价评估**
```
Step 4-1：目标买入价反推

基准目标买入价（市值口径）：
目标买入价_base = 当前市值 × (GG / II) / 总股本
  含义：若以该价格购入，穿透回报率恰好等于门槛 II%

现金保护调整后目标买入价（若适用 EV 口径）：
目标买入价_EV = (广义净现金 / 总股本) + (EV × GG_EV / II) / 总股本
  含义：净现金部分按面值计入 + 经营业务部分按门槛回报率折算

安全边际折扣调整：
- 无保护 → 目标买入价 = 目标买入价_base × 0.70
- 轻度保护 → × 0.75
- 强保护 → × 0.80
- 极强保护 → × 0.85

Step 4-2：当前股价位置评估
当前股价 = P_current
目标买入价 = P_target
距离目标买入价距离 = (P_target - P_current) / P_current × 100%

Step 4-3：买入触发条件
- 立即买入：当前股价 ≤ 目标买入价
- 观察等待：当前股价 > 目标买入价
  - 距离<5% → 高优先级观察
  - 距离 5-15% → 中优先级观察
  - 距离>15% → 低优先级观察
```

## 注意事项

1. **职责边界**：主代理只做调度和分析，不调用数据采集工具；数据采集完全由子代理完成
2. **工具分工**：严格按数据类型选用工具（见可用工具清单中的适用场景）
3. **数据优先级**：
   - 无风险利率 → mx_macro_data 优先
   - 公告/研报 → mx_finance_search 优先
   - 定性信息/管理层/竞争格局 → mx_financial_assistant 优先
   - 实时行情/股息 → mx_finance_data 优先
   - 三大报表/K线 → AI-Tools MCP 优先
4. **并行执行**：Phase 1 和 Phase 2 子代理可以并行执行，主代理等待两个文件都生成后再继续
5. **数据标注**：所有数据必须标注来源（工具名称/搜索关键词）
6. **单位统一**：报告输出使用人民币亿元，子代理数据包使用报表原始币种百万元
7. **否决纪律**：因子 1A 或因子 2 触发否决 → 立即停止后续分析
8. **策略目录只读**：不得修改 V1.x.md 历史版本文件
9. **框架边界**：不适用于 Pre-profit 企业、高增长科技股、周期顶部的强周期股

## 关键变量索引

### 因子 2 变量
A=集团净利润, B=少数股东损益, C=归母净利润, D=折旧摊销, E=资本开支,
G=维持性资本开支系数, H=D×G（维持性资本开支粗估）, I=C+D-H（Owner Earnings）,
M=支付率锚定值, O=年均回购金额, Q=综合股息税率, R=粗算穿透回报率

### 因子 3 变量
S=营业收入, T=应收账款净变动, U=预收款/合同负债净变动,
V1=资产处置收入（保留项）, V_deduct=应扣除非经常性流入,
W=经营性现金支出, Y=总扣除额, AA=真实可支配现金结余基准值,
BB=广义现金合计, FF=可自由支配现金, GG=精算穿透回报率, HH=R-GG（粗算偏差）

### 因子 4 变量
GG=精算穿透回报率, II=门槛值, JJ=GG-II（安全边际）, KK=修正后安全边际,
NN=历史价格数据点数量, OO=5年最低价, PP=5年最高价

## 特殊规则

### 合同负债计入类现金

适用条件（须同时满足）：
- 业务模式为"先款后货"或"预收制"
- 合同负债对应交付确定性极高（>95%）
- 合同负债近3年波动率 < 30%

计入逻辑：
- 用于：安全垫评估、EV 口径计算、派息可持续年限
- 不用于：可自由分配现金 FF 的计算

### EV 口径双轨制

触发条件：广义净现金/市值 > 40%

计算：
- EV = 市值 - 广义净现金
- R_EV = [C × M × (1-Q%) + O] / EV
- GG_EV = [AA × M × (1-Q%) + O] / EV

通过条件：R_EV ≥ II **且** 通过"现金分配意愿检验"

### 现金保护等级

| 等级 | 广义净现金/市值 | 安全边际折扣 |
|------|--------------|------------|
| 无保护 | < 20% | 标准 30% |
| 轻度保护 | 20-40% | 25% |
| 强保护 | 40-60% | 20% |
| 极强保护 | > 60% | 15% |

## 异常处理

| 异常情况 | 处理方式 |
|---------|---------|
| Phase 1 子代理超时 | 主代理重新启动，最多3次；仍失败则通知用户 |
| mx-* skill 调用失败 | 降级至对应备用工具（见工具清单） |
| AI-Tools 财报数据不足5年 | 使用 mx_finance_data 补充，标注实际覆盖年份 |
| data_pack_report.md 缺失 | 使用降级方案（data_pack_market.md 单独分析），标注"深度数据缺失" |
| 因子 1A/1B 触发否决 | 停止后续分析，输出否决报告 |
| 因子 2 粗算否决 | 不进入因子 3，直接输出否决结论 |
