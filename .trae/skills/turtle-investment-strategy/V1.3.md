# 龟龟投资策略分析助手

## 技能描述

本技能实现龟龟投资策略 v1.1 框架，通过**主代理+子代理**的多代理架构对股票进行深度价值分析。适用于：
- 用户请求分析某只股票
- 用户询问是否应该买入某只股票
- 用户需要估值分析和安全边际评估

## 架构设计

### 主代理（Coordinator Agent）职责
- 输入解析与任务调度
- 创建输出目录结构
- **并行启动子代理**（Phase 1 和 Phase 2）
- 监控子代理执行状态
- 整合子代理输出的数据包
- 执行因子分析（Phase 3）
- 生成最终分析报告

### 子代理（Worker Agents）职责
- **Phase 1 子代理**：市场数据采集
- **Phase 2 子代理**：深度财务数据采集
- 独立执行数据采集任务
- 输出标准化数据包文件
- 标注数据来源和缺失项

### 工作流程图
```
用户请求
    ↓
主代理：输入解析
    ↓
主代理：创建输出目录
    ↓
主代理：并行启动子代理
    ├─→ Phase 1 子代理：市场数据采集 → data_pack_market.md
    └─→ Phase 2 子代理：财务数据采集 → data_pack_report.md
    ↓
主代理：等待子代理完成
    ↓
主代理：读取数据包
    ↓
主代理：执行因子分析（Phase 3）
    ↓
主代理：生成最终报告
```

## 触发条件

当用户输入包含以下任一情况时，立即调用本技能：
1. 股票代码或名称（如"分析平安银行"、"000001"、"长和"）
2. 询问估值、安全边际、是否值得买入等问题
3. 提到"龟龟投资策略"或"turtle strategy"

## 主代理执行流程

### 第一阶段：输入解析

从用户消息中提取：
1. **股票代码/名称**：识别目标股票
2. **持股渠道**：港股通/直接持有/美股券商（未指定则用默认值）
3. **目标年份确定**：
   - 当前月份 ≤ 3 月 → target_year = 当前年份 - 2
   - 当前月份 ≥ 4 月 → target_year = 当前年份 - 1

### 第二阶段：创建输出目录

```
创建目录：{workspace}/龟龟投资策略分析报告/{symbol}/
```

### 第三阶段：并行启动子代理

**主代理同时启动两个子代理**：

#### 子代理 1：Phase 1 市场数据采集

**任务描述**：采集市场数据、财务报表、管理层信息、行业竞争等数据

**输入参数**：
```json
{
  "symbol": "股票代码",
  "company_name": "公司名称",
  "holding_channel": "持股渠道",
  "target_year": 目标年份,
  "output_path": "{workspace}/龟龟投资策略分析报告/{symbol}/data_pack_market.md"
}
```

**输出文件**：`data_pack_market.md`

**采集清单**（详见子代理规范）：
1. 基础市场数据（股价、市值、股息率等）
2. 5 年三大报表（损益表、资产负债表、现金流量表）
3. 股息与回购历史
4. 5 年历史月线价格
5. 无风险利率
6. 上市结构与税务信息
7. 管理层与治理信息
8. 行业与竞争格局
9. 子公司数据（控股公司适用）
10. MD&A 摘要

---

#### 子代理 2：Phase 2 深度财务数据采集

**任务描述**：按优先级获取深度财务数据，包括母公司单体报表、附注信息、审计报告等

**输入参数**：
```json
{
  "symbol": "股票代码",
  "company_name": "公司名称",
  "target_year": 目标年份,
  "output_path": "{workspace}/龟龟投资策略分析报告/{symbol}/data_pack_report.md"
}
```

**输出文件**：`data_pack_report.md`

**数据采集策略**：四级优先级（详见子代理规范）
- 第一优先级：AI-Tools MCP 工具
- 第二优先级：WebSearch 补充
- 第三优先级：发现报告 MCP 工具
- 第四优先级：用户上传的 PDF

---

### 第四阶段：监控子代理状态

主代理定期检查子代理输出文件：
- 检查 `data_pack_market.md` 是否生成
- 检查 `data_pack_report.md` 是否生成
- 若任一文件缺失 → 等待或重新启动子代理

### 第五阶段：数据包整合

主代理读取子代理输出的数据包：
1. 读取 `data_pack_market.md`
2. 读取 `data_pack_report.md`
3. 验证数据完整性
4. 标注缺失项

### 第六阶段：执行因子分析（Phase 3）

**任务描述**：基于数据包执行四因子分析，生成最终报告

**输入**：
- `data_pack_market.md`（必有）
- `data_pack_report.md`（可选）
- 策略知识库（phase3_分析与报告.md）

**输出**：`{workspace}/龟龟投资策略分析报告/{symbol}/{公司名}_{代码}_分析报告.md`

**分析框架**：

#### 因子 1A：五分钟快筛（一票否决）

检查 6 项：
1. 审计意见异常
2. 频繁更换审计师
3. 财务造假或重大违规前科
4. 看不懂（无法理解商业模式）
5. 商业模式未被验证
6. 控股股东重大负面信号

任一项为"是" → 否决，停止全部后续分析

#### 因子 1B：深度定性分析

**模块〇：三大报表基础数据与关键指标（强制步骤）**

**⚠️ 核心原则：数据只取一次，来源只标一次，后续因子直接引用不再重新取数**

必须完成的子任务：
1. **利润表核心数据**（逐年列表，标注来源）
   - 营业收入、营业成本、毛利润
   - 研发费用、销售费用、管理费用
   - 经营利润、其他收入/支出、税前利润
   - 所得税、归母净利润、少数股东损益
   - SBC（股权激励费用）、D&A（折旧摊销）

2. **关键盈利能力指标**
   - 毛利率、经营利润率、净利率
   - 销售费用率、研发费用率、管理费用率
   - 有效税率

3. **资产负债表核心数据**
   - 现金及等价物、短期投资、广义现金合计
   - 应收账款、存货、流动资产
   - 长期投资、商誉、无形资产、总资产
   - 短期借款、长期借款、有息负债合计
   - 预收/递延收入（合同负债）、股东权益

4. **现金流量表核心数据**
   - OCF（经营活动现金流）
   - Capex（资本开支）
   - FCF = OCF - Capex
   - 股息支付金额、回购金额

5. **因子 2/3/4 计算参数预提取（必须完成）**
   ```
   ════ 市值与股价参数 ════
   当前股价 = [X] [币种]
   总股本（稀释后）= [X] 股
   当前市值 = [X] 亿元
   汇率（若需币种换算）= [X]
   
   ════ 盈利参数 ════
   集团净利润 [A] = [值] 亿元
   少数股东损益 [B] = [值] 亿元
   GAAP 归母净利润 [C] = A − B = [值] 亿元
   主营经营利润（锚定口径）= [值] 亿元
   折旧摊销 [D] = [值] 亿元
   资本开支 [E] = [值] 亿元
   SBC = [值] 亿元
   
   ════ 现金与负债参数 ════
   现金及等价物 = [值] 亿元
   受限现金 [CC] = [值] 亿元
   短期投资 = [值] 亿元
   持有至到期定期存款/理财 = [值] 亿元
   狭义现金合计 [BB_narrow] = [值] 亿元
   广义现金合计 [BB] = [值] 亿元
   合同负债/预收款 [BB_contract] = [值] 亿元
   超广义现金 [BB_super] = [值] 亿元
   有息负债合计 = [值] 亿元
   狭义净现金 = [值] 亿元
   广义净现金 = [值] 亿元
   
   ════ 分配参数 ════
   最新年报股息总额 = [值] 亿元 [报表币种]
   最新年报归母净利润 = [值] 亿元 [报表币种]
   最新年报支付率 = [值]%
   过去 3 年支付率序列 = [X%, X%, X%]
   支付率均值 [M_hist] = [值]%
   最新派息信号 = [无变化 / DPS 已提升]
   年均回购金额 [O] = [值] 亿元
   支付率锚定值 [M] = [值]%
   
   ════ 估值基准参数 ════
   Rf（十年期国债）= [值]%
   门槛 [II] = max([X]%, Rf+[X]%) = [值]%
   ```

**后续模块**：
- 模块一：资本消耗强度
- 模块二：收款模式
- 模块三：竞争格局与护城河（两层递进：非技术性 + 技术性）
- 模块四：周期性
- 模块五：人力资本依赖
- 模块六：管理层与治理结构
- 模块七：监管与政策风险
- 模块八：MD&A 解读
- 模块九：控股折价分析（控股公司适用）

输出定性参数和定量参数，传递至因子 2/3/4

#### 因子 2：穿透回报率粗算（Top-Down）

计算步骤：
1. 归母净利润
2. Owner Earnings 粗算
3. 分配能力验证（一票否决）
4. 现金上游障碍
5. 分配意愿
6. 可预测性
7. 税务泄漏修正
8. 粗算穿透回报率 R%

**粗算否决门**：
- 若 R < Rf → 检查 EV 口径，若仍不通过则否决
- 若 R < II×0.5 → 检查 EV 口径，若仍不通过则否决
- 若 II×0.5 ≤ R < II → 标注"边际不达标"，继续进入因子 3

#### 因子 3：穿透回报率精算（Bottom-Up）+ 现金质量审计

**核心原则**：极端保守假设，逐行追踪现金流底层

**计算步骤**：

**步骤 1：真实现金收入还原**
```
营业收入 = S
应收账款净变动 = T（增加为正，减少为负）
预收款/合同负债净变动 = U（增加为正）

保守处理：
- 应收账款净增加 → 扣除（收入未收到现金）
- 预收款净增加 → 不加回（对应未来交付义务）
- 例外：高确定性预收模式（白酒/物业）可将净增加加回

真实现金收入 = S - T 的净增加部分 - U 的净减少部分
收款比率 = 真实现金收入 / S
```

**步骤 2：应收账款 footnote 核查**（收款比率<1 时必做）
- 一年以上应收占比
- 坏账计提政策 vs 同行
- 关联方应收占比
- A 股收入确认激进度判断

**步骤 3：非经常性现金流入分类**
```
保留项（真实可分配现金，不扣除）：
- 资产处置收入 V1（逐项列示）
- 其他投资收入 V5（联营分红/利息/理财收益）

扣除项（非可分配，剔除）：
- 政府补贴 V2
- 保险赔款 V3
- 其他一次性非投资类流入 V4

应扣除合计 = V2 + V3 + V4
```

**步骤 4：经营性现金支出还原**
```
供应商支付 = 营业成本 + 应付账款净减少
员工支付 = 现金流量表中支付员工现金
现金税款 = 利润表所得税 − 递延所得税变动
现金利息 = 财务费用中实际现金支付部分
  - 若资本化利息>30% → 还原为当期支出

经营性现金支出合计 = W
```

**步骤 5：资本开支与投资的极端保守处理**
```
核心原则：全部纳入扣除项，不区分维持性/成长性

(a) 资本开支总额 = E（全额扣除）
(b) 投资性购买 = X1（对外投资 + 收购 + 参股，逐项列示）
(c) 隐性必要支出还原 = X2
    - 研发费用资本化还原
    - 销售费用资本化还原
    - 递延资产费用化还原

总扣除额 Y = E + X1 + X2
```

**步骤 6：准则差异核查**
- A 股：收入确认激进度、递延税资产趋势、资本化利息比例
- 港股：HKFRS 9 预期信用损失、HKFRS 16 表外租赁
- 或有负债：对外担保、重大诉讼、资本承诺

**步骤 7：真实可支配现金结余**
```
真实可支配现金结余 = 真实现金收入
                    + 资产处置收入 V1
                    + 其他投资收入 V5
                    - 应扣除非经常性流入 V_deduct
                    - 经营性现金支出 W
                    - 总扣除额 Y

逐年列表计算
历史均值 AA = [值] 亿元
  - 若存在"非经营收入主导型年份"（保留项>经营现金流 50%）
  - 区分 AA_incl（含全部年份）和 AA_excl（剔除异常年份）
  - 两者差异>30% → 以 AA_excl 为基准
```

**步骤 8：现金储备质量验证**
```
账面现金（广义）BB = [值] 亿元
  - 狭义现金 BB_narrow = 现金 + 短投
  - 定期存款/理财 BB_deposit = 广义 − 狭义
  - 合同负债 BB_contract（类现金，仅用于安全垫和 EV 计算）

受限资金 CC = 质押/冻结/监管账户
上游障碍现金 DD = 内地子公司留存
专项用途资金 EE

可自由支配现金 FF = BB − CC − DD − EE
  - 默认使用狭义口径 FF_narrow
  - 可升级至广义口径的条件：
    (a) 定期存款剩余期限 ≤ 1 年
    (b) 无质押/冻结限制
    (c) BB_deposit / BB_narrow < 50%

派息可持续年限 = FF / 年派息总额
```

**步骤 9：派息后现金储备净变动**
```
逐年计算：
期末可自由支配现金 - 期初可自由支配现金 - 当年结余 + 当年派息及回购
判断：是否持续为正（造血>分配）
```

**步骤 10：精算穿透回报率计算**
```
选用基准值 AA = [AA_incl / AA_excl] 亿元
支付率锚定值 M = [值]%（与因子 2 同口径）

─── 主轨：市值口径 ───
精算穿透回报率 GG = [AA × M × (1 − Q%) + O] / 当前市值

─── 辅轨：EV 口径（仅当广义净现金/市值 > 40% 时）───
EV = 当前市值 − 广义净现金
精算穿透回报率 GG_EV = [AA × M × (1 − Q%) + O] / EV

决策口径判断：
- 若 GG ≥ II → 使用市值口径
- 若 GG < II 但 GG_EV ≥ II 且通过"现金分配意愿检验" → 使用 EV 口径
```

**步骤 11：收入敏感性分析**
```
经营杠杆系数 λ = 过去 3 年Δ结余/Δ收入的中位数
λ可靠性校验：
  - 收入最大波幅<10% → 可靠性低
  - Δ结余/Δ收入符号不一致 → 稳定性差
  - λ>3 或 λ<0 → 异常

收入情景分析：
| 情景 | 收入 | 预估结余 | 精算穿透回报率 | vs 门槛 | 门槛对应股价 |
| 1.0× | S | AA | GG | ±X pct | P_th |
| 0.9× | S×0.9 | AA' | GG' | ±X pct | P_th' |
| 0.8× | S×0.8 | AA'' | GG'' | ±X pct | P_th'' |
| 0.7× | S×0.7 | AA''' | GG''' | ±X pct | P_th''' |

临界收入倍数 = 临界收入 / S_current
  - ≥0.85 → 安全边际对收入波动敏感
  - 0.7-0.85 → 韧性中等
  - <0.7 → 韧性强
```

**步骤 12：与因子 1 交叉验证 + 外推可信度评级**
```
5 维度评定：
① 收入波动率（5 年 CV）：<10%高，10-25%中，>25%低
② 利润口径调整幅度：C_adj/C_rep偏差<5%高，5-15%中，>15%低
③ 粗算偏差|HH|：<1pct 高，1-3pct 中，>3pct 低
④ 经营模式变化：无重大变化高，有调整中，转型中低
⑤ λ可靠性：正常高，有一项警告中，多项警告低

评定规则：
- 5 项中≥4 项为"高" → 外推可信度 = 高
- 任意 2 项为"低" → 外推可信度 = 低
- 其余 → 外推可信度 = 中
```

#### 因子 4：估值与安全边际

**步骤 1：门槛计算**
```
Rf = [输入值]%

港股门槛 = max(5%, Rf + 3%) = [II]%
A 股门槛 = max(3.5%, Rf + 2%) = [II]%
美股门槛 = max(5%, Rf_US + 3%) = [II]%

─── 双轨门槛比较 ───
主轨（市值口径）：GG vs II
辅轨（EV 口径）：GG_EV vs II（若适用）

判断：
- GG ≥ II → 主轨通过
- GG < II 但 GG_EV ≥ II 且通过分配意愿检验 → EV 口径通过
- GG < II 且 GG_EV < II → 不达标，排除
```

**步骤 2：价值陷阱排查**（5 项检查）
```
逐项检查（调用前序因子结论）：

| 序号 | 陷阱特征 | 判断标准 | 结果 |
| 1 | 现金流趋势性恶化 | 因子 3 结余序列连续≥2 年下降且降幅>15% | [是/否] |
| 2 | 护城河正在收窄 | 因子 1B 模块三判定为"负面" | [是/否] |
| 3 | 行业结构性衰退 | 因子 1B 模块四终端需求不可逆萎缩 | [是/否] |
| 4 | 分配意愿存疑 | 因子 2 分配意愿判定为"弱" | [是/否] |
| 5 | 管理层损害价值 | 因子 1B 模块六判定为"损害价值" | [是/否] |

存在项数 N：
- N = 0 → 风险低
- N = 1 → 风险中
- N ≥ 2 → 风险高
  - 若 GG > II × 1.5 → 保留（回报率足够覆盖风险）
  - 否则 → 排除
```

**步骤 3：安全边际与仓位**
```
决策口径选择：
- 若因子 3 决策口径 = EV 口径 → 使用 GG_EV
- 否则 → 使用 GG

安全边际 JJ = 选用回报率 − II
周期性修正：
- 强周期底部 → 门槛下调 1pct
- 强周期顶部 → 门槛上调 2pct
- 非周期/周期中段 → 不调整

修正后安全边际 KK

现金保护等级判定：
- 广义净现金/市值 < 20% → 无保护（30% 折扣）
- 20-40% → 轻度保护（25% 折扣）
- 40-60% → 强保护（20% 折扣）
- > 60% → 极强保护（15% 折扣）

仓位矩阵：
| 条件 | 仓位 |
|------|------|
| 安全边际≥1.5pct + 外推可信度高 + 陷阱风险低 | 标准仓位 |
| 安全边际≥1.5pct + 外推可信度高 + 陷阱风险中 | 70% |
| 安全边际≥1.5pct + 外推可信度中 + 陷阱风险低 | 70% |
| 安全边际 0.5-1.5pct + 外推可信度高 + 陷阱风险低 | 50% |
| 安全边际 0.5-1.5pct + 外推可信度中 + 陷阱风险低 | 观察仓位（30%） |
| 安全边际 0-0.5pct | 观察仓位或等待 |
| 安全边际 < 0 | 不建仓（现金保护豁免除外） |
| 外推可信度低 | 观察仓位或不建仓 |
| 陷阱风险高且回报率<门槛×1.5 | 排除 |

现金保护豁免（仅当使用 EV 口径且安全边际≥0 时）：
- 若现金保护等级为"强保护"或"极强保护"，且市值口径安全边际<0
- → 升级为"高优先级观察"
- → 标注："EV 口径达标，等待市值口径收窄至达标"
```

**步骤 4：股价位置与买入触发价评估**
```
Step 4-1：目标买入价反推

基准目标买入价（市值口径）：
目标买入价_base = 当前市值 × (GG / II) / 总股本
  含义：若以该价格购入，穿透回报率恰好等于门槛 II%

现金保护调整后目标买入价（若适用 EV 口径）：
目标买入价_EV = (广义净现金 / 总股本) + (EV × GG_EV / II) / 总股本
  含义：净现金部分按面值计入 + 经营业务部分按门槛回报率折算

安全边际折扣调整：
- 无保护 → 目标买入价 = 目标买入价_base × 0.70
- 轻度保护 → × 0.75
- 强保护 → × 0.80
- 极强保护 → × 0.85

Step 4-2：当前股价位置评估
当前股价 = P_current
目标买入价 = P_target
距离目标买入价距离 = (P_target - P_current) / P_current × 100%

Step 4-3：买入触发条件
- 立即买入：当前股价 ≤ 目标买入价
- 观察等待：当前股价 > 目标买入价
  - 距离<5% → 高优先级观察
  - 距离 5-15% → 中优先级观察
  - 距离>15% → 低优先级观察
```

## 主代理与子代理交互协议

### 主代理启动子代理的方式

主代理通过以下方式启动子代理：

1. **创建子代理任务描述**
   ```
   主代理创建任务描述文件，包含：
   - 任务类型：Phase 1 或 Phase 2
   - 输入参数：symbol, company_name, target_year, output_path
   - 采集清单：详细的数据项列表
   - 数据源优先级：AI-Tools > WebSearch > 发现报告 > 用户上传
   - 输出格式：标准化的 markdown 文件
   ```

2. **并行启动子代理**
   ```
   主代理同时启动 Phase 1 和 Phase 2 子代理
   子代理独立执行，互不依赖
   ```

3. **监控子代理状态**
   ```
   主代理定期检查输出文件是否生成：
   - 检查 data_pack_market.md
   - 检查 data_pack_report.md
   - 若文件存在 → 读取并验证数据完整性
   - 若文件缺失 → 等待或重新启动子代理
   ```

4. **整合子代理输出**
   ```
   主代理读取两个数据包文件
   验证数据完整性
   标注缺失项
   将数据传递给因子分析模块
   ```

### 子代理输出文件规范

子代理必须输出标准化的 markdown 文件，包含：
1. 文件头部：数据采集时间、数据来源标注、金额单位
2. 数据主体：按章节组织的数据项
3. 数据来源：每个数据项标注来源
4. 缺失项列表：标注未找到的数据项
5. 数据质量评估：完整度和可靠性评估

### 错误处理机制

1. **子代理失败**
   - 主代理检测到子代理输出文件缺失
   - 主代理重新启动子代理
   - 若重试 3 次仍失败 → 标注"数据采集失败"，继续分析

2. **数据缺失**
   - 子代理在输出文件中标注缺失项
   - 主代理在分析报告中标注"⚠️数据缺失"
   - 主代理使用保守估计或跳过相关分析

3. **数据冲突**
   - 若多个数据源数据不一致 → 主代理优先使用 AI-Tools 数据
   - 标注数据冲突情况
   - 在分析报告中说明数据来源选择

## 注意事项

1. **数据来源优先级**
   - 始终遵循四级优先级：AI-Tools > WebSearch > 发现报告 > 用户上传
   - 标注每个数据项的来源

   - 子代理必须标注缺失项
   - 主代理在分析报告中说明数据缺失的影响

3. **并行执行**
   - Phase 1 和 Phase 2 子代理可以并行执行
   - 主代理等待两个子代理都完成后再执行 Phase 3

4. **错误处理**
   - 子代理失败时主代理应重试
   - 数据缺失时主代理应使用保守估计

5. **单位统一**
   - 所有金额单位为报表币种百万元
   - 必须在文件头部标注单位信息

6. **数据时效性**
   - 优先使用最新财年数据
   - 标注数据截止日期
