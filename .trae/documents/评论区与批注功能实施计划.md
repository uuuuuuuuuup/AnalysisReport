# 评论区与批注功能实施计划

## 功能概述

为龟龟投资策略博客添加两个核心互动功能：
1. **评论区** - 用户可以对每只股票的分析报告发表评论
2. **批注功能** - 用户可以在报告内容上添加个人批注和高亮标注

## 技术方案

### 数据存储方案
- **使用 LocalStorage** - 无需后端，数据存储在用户浏览器本地
- **数据结构** - JSON 格式存储评论和批注
- **数据关联** - 通过 `股票代码 - 模型名称` 唯一标识关联数据

### 技术栈
- **原生 JavaScript** - 与现有代码保持一致
- **CSS 样式** - 扩展现有样式系统
- **marked.js** - 用于评论内容的 Markdown 渲染（已集成）


## 潜在挑战与解决方案

### 挑战 1: LocalStorage 容量限制
**问题**: LocalStorage 只有 5-10MB 容量限制
**解决方案**: 
- 限制评论长度
- 定期清理旧数据
- 数据压缩存储
- 考虑 IndexedDB（如需要）

### 挑战 2: 批注位置定位
**问题**: 内容变化后批注位置可能偏移
**解决方案**:
- 使用文本内容定位而非索引
- 批注关联到章节而非具体位置
- 检测内容变化并提示用户

### 挑战 3: 跨模型数据隔离
**问题**: 不同模型的报告需要独立的评论和批注
**解决方案**:
- 使用 `stockCode-modelName` 组合作为键
- 切换模型时重新加载数据
- 清晰的数据标识

### 挑战 4: Markdown 渲染与安全
**问题**: 用户输入的 Markdown 可能包含恶意脚本
**解决方案**:
- 使用 DOMPurify 过滤 HTML
- 限制 Markdown 语法范围
- 转义脚本标签
